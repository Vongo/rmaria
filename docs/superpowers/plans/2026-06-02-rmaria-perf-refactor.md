# rmaria Performance & Safety Refactor — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the critical SQL-injection, data-corruption, connection-per-row, silent-row-loss, and data.table-crash defects in `rmaria`'s SELECT/INSERT/UPSERT/UPDATE/DELETE wrappers without changing public signatures, then (Phase 2) restructure for performance and maintainability.

**Architecture:** The package builds SQL by string concatenation and opens a fresh DB connection per statement. Phase 1 keeps the existing function shapes but (a) opens **one** connection per call and reuses it, (b) replaces the broken hand-rolled escaping with DBI's `dbQuoteLiteral`/`dbQuoteIdentifier`, (c) normalizes inputs to `data.frame`, (d) makes chunk sizing integer, and (e) guarantees disconnect via `on.exit`. Phase 2 (separate plan) introduces shared helpers (`resolve_credentials`, `with_connection`/`pool`), vectorized/batched writes (`dbAppendTable`, multi-row upsert, temp-table-join update), transactions, and packaging hygiene.

**Tech Stack:** R, RMariaDB, DBI, data.table, purrr, bit64, pool (Phase 2), testthat (+ dockerized MariaDB for integration), roxygen2.

**Decisions locked (2026-06-02):** Two-phase (fixes → refactor) · keep signatures, fix behavior · declare `purrr`/`bit64`, use DBI quoting, add `pool` (Phase 2) · dockerized MariaDB integration tests in CI + pure unit tests.

**Source of findings:** Adversarial multi-agent review (74 agents, 56 confirmed findings) reconciled with empirical R 4.6 checks. Severity tags reference the review themes (A=injection/escaping, B=performance, C=data.table, D=robustness, E=packaging, F=consistency).

## Conventions used by every task

- **Internal helpers** are prefixed `.` (e.g. `.maria_connect`) and are **not** exported.
- **Quoting:** values via `DBI::dbQuoteLiteral(con, x)` (handles type + `NA` → `NULL`, vectorized); identifiers via `DBI::dbQuoteIdentifier(con, x)` (backtick-quoted).
- **Charset:** `SET NAMES utf8mb4` once per connection (replaces `set character set "utf8"`).
- **Cleanup:** `on.exit(RMariaDB::dbDisconnect(con), add = TRUE)` immediately after connecting.
- **Tests** that need a database start with `skip_if_no_db()` so the suite passes locally without one.

```r
test_con <- function() {
  e <- db_env()
  con <- RMariaDB::dbConnect(RMariaDB::MariaDB(), dbname = e$db, host = e$host,
                             port = e$port, user = e$user, password = e$pwd)

  RMariaDB::dbExecute(con, create_sql)
  on.exit(RMariaDB::dbExecute(con, sprintf("DROP TABLE IF EXISTS `%s`", table_name)), add = TRUE)
  force(code)
```

```r
insert_table <- function(table, table_name_in_base, host="localhost", port=3306, db, user, password, chunk_size=NA, progress_bar=TRUE, ignore=TRUE, nolog=FALSE, allow.backslash=FALSE) {
  init()
  table <- as.data.frame(table)                                  # C1: data.table-safe
  if (nrow(table) == 0) {
    if (!nolog) logging::logwarn("You tried to insert an empty table. Leaving.", logger=LOGGER.MAIN)
    return(invisible())
  }
  if (!nolog) logging::loginfo("Inserting data into table %s.", table_name_in_base, logger=LOGGER.MAIN)
  if (is.na(chunk_size)) {
    chunk_size <- max(min(10000L, nrow(table) %/% 25L), 100L)     # D2: integer
  }
  chunk_size <- as.integer(min(chunk_size, nrow(table)))
  n_iter <- ceiling(nrow(table) / chunk_size)

  con <- .maria_connect(host, port, db, user, password)          # B2/D1: one connection
  on.exit(RMariaDB::dbDisconnect(con), add = TRUE)

  tbl_sql  <- DBI::dbQuoteIdentifier(con, table_name_in_base)     # A2
  cols_sql <- paste(DBI::dbQuoteIdentifier(con, colnames(table)), collapse = ",")
  pb <- if (progress_bar) create_pb(n_iter, bar_style="pc", time_style="cd") else NULL

  for (i in seq_len(n_iter)) {
    rows <- ((i - 1L) * chunk_size + 1L):min(i * chunk_size, nrow(table))
    # Quote each column vectorised (NA -> NULL, type-correct), then assemble tuples. A1/D8/B3.
    quoted <- lapply(table[rows, , drop = FALSE],
                     function(col) as.character(DBI::dbQuoteLiteral(con, col)))
    tuples <- do.call(paste, c(quoted, sep = ","))
    values_sql <- paste0("(", tuples, ")", collapse = ",")
    query <- paste0("INSERT ", if (ignore) "IGNORE " else "", "INTO ", tbl_sql,
                    " (", cols_sql, ") VALUES ", values_sql)
    tryCatch(
      RMariaDB::dbExecute(con, query),
      error = function(e) if (!nolog) logging::logerror(
        "Error while inserting chunk %d/%d: %s", i, n_iter, conditionMessage(e), logger=LOGGER.MAIN)
    )
    if (progress_bar) update_pb(pb, i)
  }
  invisible()
}
```

- The old NA sentinel (`"Qù@ñ…"`), `has_quotes`, the no-op `gsub` chain, and the per-chunk `exec_query` call are all removed — `dbQuoteLiteral` replaces them.
- The `allow.backslash` parameter is retained in the signature for compatibility but is now a no-op (DBI escapes backslashes correctly); document this in Step 4.

```r
      if (is.na(table[i, ic])) return("")
      paste0(DBI::dbQuoteIdentifier(con, colnames(table)[ic]), "=", vals[ic])
    }, character(1))
    set_parts <- set_parts[nzchar(set_parts)]
    if (length(set_parts) == 0L) {
      # nothing to update on conflict: fall back to INSERT IGNORE semantics
      query <- paste0("INSERT IGNORE INTO ", tbl_sql, " (", cols_sql, ") VALUES ", values_sql, ";")
    } else {
      query <- paste0("INSERT INTO ", tbl_sql, " (", cols_sql, ") VALUES ", values_sql,
                      " ON DUPLICATE KEY UPDATE ", paste(set_parts, collapse = ","), ";")
    }
    tryCatch(
      RMariaDB::dbExecute(con, query),
      error = function(e) if (!nolog) logging::logerror(
        "Error while upserting row %d: %s", i, conditionMessage(e), logger=LOGGER.MAIN)
    )
    if (progress_bar) update_pb(pb, i)
  }
  invisible()
```

Note: this keeps the row-at-a-time statement loop (Phase 1) but on a **single** connection (the catastrophic per-row connect/disconnect is gone). Batching into one multi-row statement is Phase 2.

# Phase 2 — Structural & performance refactor (separate plan)

> A detailed bite-sized plan will be written (via writing-plans) **after PR #1 merges**, because its tasks build on the helpers introduced above. Captured here as a roadmap so nothing is lost.

**P2-1 — Extract shared helpers.** `resolve_credentials()` replacing the ~210 duplicated lines across the 7 `*q` wrappers (F1); harden the env-scan or replace with an explicit credentials object/config (F2). Split `R/maria.R` into `R/credentials.R`, `R/connection.R`, `R/sql-builders.R`, `R/select.R`, `R/write.R` (coding-style: many small files).

**P2-2 — Pure, testable SQL builders.** `build_insert_sql()`, `build_upsert_sql()`, `build_update_plan()` returning parameterized SQL + params; unit-test with no DB (`test-sql-builders.R`).

**P2-3 — Connection layer / pooling.** `with_connection(creds, fn)` and the `pool` package for reuse across operations (B5); wrap multi-statement writes in `dbWithTransaction` (D4).

**P2-4 — Vectorize & batch the hot paths.**
- `insert_table` → `DBI::dbAppendTable` (chunked) (B3, B4).
- `upsert_table` → one multi-row `INSERT … ON DUPLICATE KEY UPDATE col = new.col` per chunk using the `AS new` alias (MySQL 8.0.20+; keep `VALUES()` fallback for MariaDB) — collapses N statements to N/chunk (B1, B3).
- `update_table` → load batch into a `TEMPORARY` table and run one `UPDATE target JOIN tmp …` (idiomatic high-perf bulk update) (B1, B3, B5).
- Parameterized statements throughout (`params=`), removing all string interpolation of values (A3).

**P2-5 — Behavior polish.** Propagate errors / return affected-row counts (D5); `stop()` instead of `return(FALSE)` on missing creds (F3); unify `progress_bar` default to `interactive()` (F7); redact data values from `logfinest`/`logwarn` query logs (A4).

**P2-6 — Packaging hygiene.** `@importFrom` for `map_lgl`/`%ni%`/`add`; drop `init()`/`library()` calls and the load-time `init()`; package-level `LOGGER.MAIN` constant instead of `<<-` (E2, E3). Remove dead code: `insert_source_full_file`, `esq`, `edq`, commented charset block (F6).

**P2-7 — Benchmark.** Before/after timing for a 50k-row insert, upsert, and update to quantify the connection-reuse + batching gains.
